# Vendor Slug Duplication Audit
## 19 May 2026

## Scope

Vendor slugs are generated by appending the city to the company name. When the company name already contains the city (e.g., "Cardiff Property Partners" + city "Cardiff"), the slug duplicates: `cardiff-property-partners-cardiff`.

### Known affected vendor
- Cardiff Property Partners → slug: `cardiff-property-partners-cardiff`
- Profile URL: `https://www.tendorai.com/suppliers/vendor/cardiff-property-partners-cardiff`

## Slug generation code

**File:** `models/Vendor.js:435-448`

```javascript
vendorSchema.pre('save', function(next) {
  if (!this.slug && this.company) {
    this.slug = this.company
      .toLowerCase()
      .replace(/[^a-z0-9\s-]/g, '')
      .replace(/\s+/g, '-')
      .replace(/-+/g, '-')
      .trim();
    if (this.location?.city) {
      this.slug += '-' + this.location.city     // ← ALWAYS appends city
        .toLowerCase()
        .replace(/[^a-z0-9\s-]/g, '')
        .replace(/\s+/g, '-');
    }
  }
  next();
});
```

**Bug:** Line 443-448 unconditionally appends city. No check for whether the company name already contains the city name.

## Proposed fix

### 1. Fix the slug generator (models/Vendor.js:443-448)

```javascript
// Before:
if (this.location?.city) {
  this.slug += '-' + this.location.city.toLowerCase()...;
}

// After:
if (this.location?.city) {
  const citySlug = this.location.city.toLowerCase().replace(/[^a-z0-9\s-]/g, '').replace(/\s+/g, '-');
  if (!this.slug.includes(citySlug)) {
    this.slug += '-' + citySlug;
  }
}
```

### 2. Migration script

For each affected vendor:
1. Compute clean slug (company only, or company + city without duplication)
2. Check uniqueness — if clean slug is already taken by another vendor, skip
3. Store old slug in `vendor.previousSlugs` array (new field, `[String]`)
4. Update `vendor.slug` to clean slug

```javascript
// Pseudocode
for (const vendor of affectedVendors) {
  const cleanSlug = computeCleanSlug(vendor.company, vendor.location.city);
  const taken = await Vendor.findOne({ slug: cleanSlug, _id: { $ne: vendor._id } });
  if (taken) { console.log(`SKIP: ${cleanSlug} already taken`); continue; }
  await Vendor.updateOne(
    { _id: vendor._id },
    { $set: { slug: cleanSlug }, $push: { previousSlugs: vendor.slug } }
  );
}
```

### 3. Add 301 redirect middleware

On `/suppliers/vendor/:slug` route (frontend — Next.js middleware or getServerSideProps):

```javascript
// If slug not found by primary lookup:
const vendor = await Vendor.findOne({ previousSlugs: slug });
if (vendor) {
  return redirect(301, `/suppliers/vendor/${vendor.slug}`);
}
```

### 4. Update Writer Agent stored CTAs

Search existing ApprovalQueue items for old slug URLs:

```javascript
await ApprovalQueue.updateMany(
  { 'draftPayload.body': { $regex: oldSlug } },
  // Can't do regex replace in MongoDB — would need per-document update
);
```

Simpler: let existing drafts keep old URLs (they'll 301 redirect). New drafts use the correct slug automatically.

### 5. Update sitemap

Sitemap at `/sitemap.xml` reads `vendor.slug` — automatically uses the updated slug after migration. No code change needed.

## Risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| SEO link equity loss | Medium | 301 redirects preserve equity. Google transfers ~90-99% of link equity via 301. |
| Stored CTA URLs in published posts | Low | 301 redirect covers this. Old URLs still work. |
| External inbound links to old slugs | Low | All slugs are recent (platform launched 2026). Few external links exist. |
| Unique constraint violation during migration | Low | Check uniqueness before update. Skip conflicts. |
| Frontend cache of old slugs | Low | Next.js ISR will revalidate. Clear Vercel cache after migration. |
